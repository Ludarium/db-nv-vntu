# Інструкція з розгортання бази даних it_department

Цей посібник описує процес розгортання серверної частини бази даних (MySQL) та налаштування клієнтського інтерфейсу (MS Access) для коректної роботи системи. Будь ласка, виконуйте кроки строго у зазначеній послідовності.

## Частина 1. Встановлення та налаштування СУБД MySQL

**1. Встановлення бібліотек Visual C++**
Встановіть пакет `vc_redist.x64.exe` (Microsoft Visual C++ Redistributable). Це обов'язковий компонент середовища виконання, без якого сервер MySQL та ODBC-драйвери можуть не запуститися або працювати з помилками.

**2. Встановлення MySQL Server**
Запустіть `mysql-installer-community` і при виборі типу встановлення оберіть **Full** (Повне). Ви можете знайти необхідний інсталятор за посиланням: https://dev.mysql.com/downloads/installer/ Щоб уникнути проблем із поганим інтернет з'єднанням, завантажте .msi версію. Підтверджуйте всі подальші операції за замовчуванням. У процесі налаштування придумайте та **обов'язково запам'ятайте** пароль для користувача `root` — він знадобиться для підключення. Повне встановлення гарантує, що у вас з'явиться як сам сервер, так і зручна програма MySQL Workbench для керування базою.

**3. Ініціалізація структури та даних**
Відкрийте будь-який клієнт для роботи з MySQL (наприклад, встановлений MySQL Workbench або HeidiSQL) і підключіться до локального сервера. Відкрийте та виконайте наступні `.sql` файли **строго у вказаному порядку**, щоб уникнути помилок порушення зв'язків (Foreign Keys):
*   `01_init.sql` — створення таблиць та зв'язків.
*   `02_testData.sql` — заповнення базовими та тестовими даними.
*   `03_procedures.sql` — створення збережених процедур (якщо застосовується).
*   `04_views.sql` — створення "розумних" представлень (Views) для зручного виводу в Access.
*   `05_schedules.sql` — налаштування розкладів та подій БД.

## Частина 2. Налаштування підключення (ODBC)

**4. Встановлення ODBC-конектора**
Встановіть драйвер `mysql-connector-odbc` (версія x64). Розрядність драйвера має збігатися з розрядністю вашої версії MS Office/Access та Windows.

**5. Відкриття диспетчера ODBC**
Відкрийте меню "Пуск" у Windows, введіть у пошук **Джерела даних ODBC (64-розрядна версія)** / *ODBC Data Sources (64-bit)* і запустіть програму від імені адміністратора.

**6. Створення системного підключення**
Перейдіть на вкладку **Системний DSN** (System DSN) і натисніть кнопку **Додати** (Add). Вибір системного DSN гарантує, що підключення працюватиме для всіх користувачів комп'ютера.

**7. Вибір драйвера**
У списку, що з'явиться, виберіть драйвер **MySQL ODBC X.X Unicode Driver** і натисніть "Готово". Важливо вибрати саме *Unicode*, а не *ANSI*, щоб кирилиця (український/російський текст) у базі даних відображалася коректно і без "ієрогліфів".

**8. Налаштування параметрів Data Source**
Заповніть форму підключення:
*   **Data Source Name (DSN):** Придумайте зрозуміле ім'я (наприклад, `VNTU_DB`). **Обов'язково запам'ятайте або запишіть це ім'я**, воно критично важливе для MS Access.
*   **TCP/IP Server:** Вкажіть `localhost` або `127.0.0.1`.
*   **User / Password:** Введіть дані від вашого MySQL (наприклад, `root` та пароль із кроку 2).
*   **Database:** Виберіть вашу базу з випадаючого списку (наприклад, `it_department`).

**9. Критично важливі прапорці (Details)**
У цьому ж вікні натисніть на кнопку **Details >>** (Деталі), перейдіть на вкладку **Connection** (або *Cursors/Results* залежно від версії) і **обов'язково** поставте галочки напроти:
*   `Return matched rows instead of affected rows`
*   `Allow big result sets`
*Ці налаштування необхідні, щоб MS Access не зависав під час виконання складних запитів та коректно оновлював записи без помилки "#DELETED".*

**10. Збереження**
Натисніть **Test**, щоб переконатися в успішному з'єднанні, а потім натисніть **ОК** для збереження DSN.

## Частина 3. Підключення клієнтської частини (MS Access)

**11. Запуск клієнта**
Відкрийте файл `it_department.accdb` за допомогою Microsoft Access.

**12. Запуск майстра зовнішніх даних**
На верхній панелі (стрічці) перейдіть на вкладку **Зовнішні дані** (External Data) -> **Створити джерело даних** (New Data Source) -> **З інших джерел** (From Other Sources) -> **База даних ODBC** (ODBC Database).

**13. Вибір типу зв'язку**
У вікні, що з'явиться, виберіть другий пункт: **"Створити зв'язок з таблицею-джерелом..."** (Link to the data source by creating a linked table). Це дозволить Access працювати з живими даними на сервері, а не імпортувати їх статичну копію.

**14. Прив'язка таблиць та представлень**
У вікні вибору джерела перейдіть на вкладку **Джерело даних комп'ютера** (Machine Data Source) і виберіть DSN, створений у кроці 8. З'явиться список усіх таблиць і View з MySQL:
*   Виділіть усі необхідні таблиці та представлення (`view_...`).
*   Натисніть ОК.
*   *Важливо: Якщо Access попросить вказати "Унікальний ідентифікатор запису" (Unique Record Identifier) для деяких View, обов'язково вибирайте поле `id`, щоб ці дані можна було оновлювати.*

## Частина 4. Утиліта MySQLDump для операції резервного копіювання 
**15. Безпечне зберігання облікових даних**

Щоб уникнути зберігання пароля бази даних у відкритому вигляді всередині скрипта резервного копіювання, ми використовуємо утиліту `mysql_config_editor` для створення зашифрованого профілю входу.

Відкрийте Командний рядок Windows (`cmd`) від імені Адміністратора.
Виконайте наступну команду, щоб створити профіль з іменем `backup_admin`:
```bat
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql_config_editor.exe" set --login-path=backup_admin --host=localhost --user=root --password
```
Перевірте, чи успішно створено профіль:
```bat
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql_config_editor.exe" print --all
```

**16. Скрипт резервного копіювання**

Запустіть скрипт mysql_backup.bat для генерації .sql дампа.
Примітка: скрипт автоматично генерує .sql дамп із позначкою часу.

**17. Автоматизація за допомогою Планувальника завдань**

Щоб запускати резервне копіювання автоматично (наприклад, щоночі о 23:00):
1. Відкрийте Планувальник завдань Windows (Task Scheduler) та натисніть "Створити завдання" (Create Task).

2. Вкладка "Загальні" (General): Назвіть завдання "MySQL Daily Backup". Поставте галочку "Виконувати з найвищими правами" (Run with highest privileges) та оберіть "Виконувати для всіх користувачів (незалежно від реєстрації)" (Run whether user is logged on or not).

3. Вкладка "Тригери" (Triggers): Додайте новий тригер -> Щодня -> Встановіть час (наприклад, 23:00).

4. Вкладка "Дії" (Actions): Додайте нову дію -> Запуск програми -> Натисніть "Огляд" та оберіть ваш файл mysql_backup.bat.

5. Збережіть завдання.

**18. Тестування відновлення (Тест у пісочниці)**

1. Створення тестової бази даних (пісочниці)
Відкрийте ваш MySQL-клієнт (наприклад, MySQL Workbench) і виконайте:
```sql
CREATE DATABASE test_recovery_db;
```

2. Імпорт резервної копії
Відкрийте Командний рядок Windows та відновіть дамп у нову тестову базу даних. Для тестування рекомендується використовувати користувача root, щоб обійти будь-які обмеження прав доступу, специфічні для бази даних.
```bat
"C:\Program Files\MySQL\MySQL Server 8.0\bin\mysql.exe" -u root -p test_recovery_db < "C:\MySQL_Backups\your_backup_file.sql"
```
3. Перевірка цілісності даних
Оновіть схеми баз даних у MySQL Workbench. Перевірте test_recovery_db, щоб переконатися, що всі таблиці, записи та зв'язки були відновлені коректно.

4. Очищення
Після успішної перевірки безпечно видаліть тестову базу даних:
```sql
DROP DATABASE test_recovery_db;
```
## Частина 5. Вирішення проблем (Troubleshooting)

**19. Помилка втрати зв'язку з ODBC**
Якщо під час відкриття форм або таблиць з'являється помилка підключення ODBC (особливо якщо ви перенесли базу на інший ПК), необхідно оновити зв'язки:
1. Перейдіть: **Зовнішні дані** (External Data) -> **Диспетчер зв'язаних таблиць** (Linked Table Manager).
2. Виділіть галочкою всі таблиці з іконкою глобуса.
3. Натисніть у меню праворуч "Змінити" або "Edit" та оновіть рядок підключення (ODBC Connect Str) на актуальне ім'я вашого DSN: `DSN=Назва вашого DSN;...`.
